# SiteProbe デザインレビュー（designer）

## 1. デザインシステム提案

### 1.1 カラーパレット

SiteProbeは「プロフェッショナルな分析ツール」であり、信頼感・知性・精密さを伝えるカラースキームを提案する。

#### プライマリカラー（ブランドカラー）

| 用途 | カラー名 | Hex | 説明 |
|------|----------|-----|------|
| Primary | Deep Navy | `#1e3a5f` | メインブランドカラー。信頼感・知性 |
| Primary Light | Royal Blue | `#3b82f6` | ボタン・リンク・アクティブ状態 |
| Primary Dark | Midnight | `#0f1d32` | ヘッダー・フッター背景 |

#### アクセントカラー

| 用途 | カラー名 | Hex | 説明 |
|------|----------|-----|------|
| Accent | Teal | `#14b8a6` | CTA・ハイライト・差別化要素 |
| Accent Light | Cyan | `#22d3ee` | プログレスバー・アニメーション |

#### セマンティックカラー（スコア表示用）

| スコア帯 | カラー | Hex | 意味 |

| 90-100 | Emerald | `#10b981` | 優秀 |

### 1.2 タイポグラフィ

#### フォントファミリー

```css
/* 日本語対応の推奨フォントスタック */
--font-sans: 'Inter', 'Noto Sans JP', -apple-system, BlinkMacSystemFont, sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', monospace; /* コード例表示用 */
```

- **Inter**: 英数字・UI要素に最適。Google Fontsで無料利用可能
- **Noto Sans JP**: 日本語の可読性が高く、ウェイトが豊富。Google Fontsで無料利用可能
- **JetBrains Mono**: コード例の表示用。改善提案のコードスニペットに使用

#### フォントサイズスケール

| 用途 | サイズ | ウェイト | 行間 |
|------|--------|---------|------|
| h1（ページタイトル） | 2rem (32px) | 700 (Bold) | 1.2 |
| h2（セクション見出し） | 1.5rem (24px) | 600 (SemiBold) | 1.3 |
| h3（小見出し） | 1.25rem (20px) | 600 (SemiBold) | 1.4 |
| body（本文） | 1rem (16px) | 400 (Regular) | 1.7 |
| small（補足テキスト） | 0.875rem (14px) | 400 (Regular) | 1.5 |
| caption（キャプション） | 0.75rem (12px) | 500 (Medium) | 1.4 |

**日本語本文の行間は1.7を推奨**。日本語は漢字が多く、英語より行間を広めに取ることで可読性が向上する。

## 2. ユーザー体験の設計

### 2.1 ユーザージャーニー（URL入力から結果表示まで）

```
[1. ランディング]     [2. URL入力]      [3. ペルソナ選択]    [4. 分析中]        [5. 結果表示]
  サービス説明  →   URL入力 &     →   実行ペルソナ    →   リアルタイム  →   サマリー +
  特徴紹介          バリデーション     確認/選択           プログレス         詳細レポート
  デモ結果表示                                              ストリーミング

                                        (省略可能:              |
                                         デフォルト             各ペルソナ
                                         全選択)               完了順に表示
```

#### 各ステップの詳細

**Step 1 - ランディング**:
- サービスの価値提案を3秒以内に伝える
- 「URLを入れるだけ」のシンプルさを強調
- デモ分析結果のサンプルを表示し、アウトプットイメージを提示

**Step 2 - URL入力**:
- 入力と同時にURLバリデーション（形式チェック + アクセス可能性の簡易チェック）
- プレースホルダー例: `https://example.com`
- 「分析開始」ボタンは入力が有効な場合のみアクティブ

**Step 3 - ペルソナ選択（オプション）**:
- デフォルトは全ペルソナ選択状態
- カテゴリ単位でON/OFF切り替え可能
- 個別ペルソナのON/OFF切り替え可能
- 「この手順をスキップして全ペルソナで分析」ボタンで即実行可能
- **初回ユーザーはこのステップを飛ばしてもよい設計にする**

**Step 4 - 分析中**:
- 後述の待機体験デザインを適用
- 完了したペルソナから順次結果をプレビュー表示

**Step 5 - 結果表示**:
- 総合サマリー → カテゴリ別 → 個別ペルソナ の3段階で情報を階層化

### 2.2 分析中の待機体験

分析には数分かかるため、待機体験のデザインが極めて重要。以下を提案する。

#### プログレス表示の構成

```
|  |  [v] コンテンツストラテジスト       完了       |       |
|  |  [v] SNSマーケター                完了       |       |
|  +----------------------------------------------+       |
|                                                          |
|  +-- デザイン・UX系 (3/4 分析中) ----------------+       |
|  |  [v] UIデザイナー                  完了       |       |

|  |  [v] アクセシビリティ専門家         完了       |       |

|  +----------------------------------------------+       |

|  +----------------------------------------------+       |
|                                                          |

```

#### 待機中のエンゲージメント施策

1. **完了ペルソナの結果プレビュー**: 完了したペルソナのスコアと上位3件の改善提案を即座にカード形式で表示
2. **分析中ペルソナのステータスメッセージ**: 「HTMLの構造を分析しています...」「アクセシビリティチェック中...」等のリアルタイムメッセージ
3. **パルスアニメーション**: 分析中のペルソナにはパルスアニメーション（呼吸するような明滅）を付与
4. **スケルトンUI**: 結果エリアにスケルトンローダーを表示し、完了次第実データに置換

#### アニメーション仕様

| 要素 | アニメーション | Duration | Easing |
|------|-------------|----------|--------|
| 全体プログレスバー | 左→右のスムーズ遷移 | 300ms | ease-out |
| ペルソナ完了チェック | スケールイン + フェード | 200ms | ease-out |
| 分析中インジケータ | パルス（明滅） | 1.5s | ease-in-out |
| 結果カード表示 | スライドアップ + フェードイン | 400ms | ease-out |
| スコアゲージ | 0→値のアニメーション | 800ms | ease-out |

### 2.3 分析結果の見せ方

#### 推奨: サイドバー + メインコンテンツ レイアウト

- タブ16個は多すぎて視認性が悪い
- アコーディオンは全体像の把握が困難
- カード一覧は情報の深掘りが困難

## 4. レスポンシブ対応

### 4.1 ブレイクポイント定義

| 名称 | 幅 | レイアウト |
|------|-----|----------|
| Mobile | < 640px | シングルカラム |
| Tablet | 640px - 1024px | 2カラム（簡略化） |
| Desktop | > 1024px | フルレイアウト |

### 4.2 モバイル表示方針

#### ランディングページ
- URL入力フォームはフル幅
- 特徴セクションは1カラムの縦積み
- デモ結果は横スクロール可能なカード

#### 分析結果ページ
- サイドバーは**ボトムシート（下部ドロワー）**に変換
- スワイプアップでペルソナ一覧を表示
- メインコンテンツはフル幅
- レーダーチャートはタップで拡大表示可能

```
モバイル レイアウト:
+----------------------------+
| [< 戻る] SiteProbe [共有]  |
+----------------------------+
| 総合スコア: 78/100          |
| [円形ゲージ]               |
+----------------------------+
| [マーケ][デザイン][技術]... |  ← 横スクロール可能なカテゴリタブ
+----------------------------+
|                            |
| 改善提案一覧               |
| (カード形式で縦積み)        |
|                            |
+----------------------------+
| [ペルソナ一覧 ↑]           |  ← タップでボトムシート展開
+----------------------------+
```

#### ペルソナ管理画面
- カード形式でペルソナを縦積み表示
- ON/OFFトグルは右端に配置
- 編集はフルスクリーンモーダルで開く

### 4.3 タブレット表示方針

- ランディング: デスクトップに準拠（やや余白を調整）
- 分析結果: サイドバーを折りたたみ可能な狭小版に変更（アイコンのみ表示、ホバーで展開）
- ペルソナ管理: 2カラムのグリッド表示

## 5. ブランディング

### 5.1 サービス名「SiteProbe」のビジュアルイメージ

#### ロゴコンセプト

**Probe（探査・調査）** のイメージを活かしたロゴ:

```
  [o]  SiteProbe
   ^
  虫眼鏡 or レーダーをモチーフにしたシンボルマーク
```

**ロゴ構成**:
- **シンボルマーク**: 虫眼鏡の形状にレーダーの同心円を組み合わせ。「深い調査」と「多角的分析」を表現
- **ロゴタイプ**: Inter Bold で "Site" を通常ウェイト、"Probe" を太字。Probeの「o」をシンボルマークと統合
- **カラー**: Primary Deep Navy をメインに、Accent Tealをポイントに使用

#### ロゴバリエーション

| バリエーション | 用途 |
|-------------|------|
| フルロゴ（横） | ヘッダー、マーケティング素材 |
| シンボルのみ | ファビコン、アプリアイコン |
| モノクロ | 印刷用、PDFレポートヘッダー |

### 5.2 スコア表示のビジュアル案

#### 総合スコア: ドーナツゲージ

```
      ╭─────────╮
    ╱    78      ╲
   │   /100       │    ← ドーナツ型の円形ゲージ
   │              │       スコアに応じてセマンティックカラーで塗り分け
    ╲   良好     ╱       中央に数値と評価ラベル
      ╰─────────╯
```

- ゲージの色はスコア帯に応じたセマンティックカラー
- 0→値のアニメーション（800ms ease-out）
- 中央に数値（大）+ 評価ラベル（小: 優秀/良好/改善推奨/要改善）

#### カテゴリ別スコア: レーダーチャート

```
            ╱╲
           ╱  ╲
    ユーザー╱    ╲デザイン
     (75) ╱  ★  ╲ (71)
          ╲     ╱
     ビジネス╲  ╱技術
      (68)  ╲╱  (85)
```

- 5カテゴリ（マーケ/デザイン/技術/ビジネス/ユーザー）のレーダーチャート
- 各軸のカテゴリカラーを使用
- ホバーで数値をツールチップ表示
- ライブラリ: recharts（Next.js互換、軽量）を推奨

#### 改善提案の優先度バッジ

```
  [!   低]  グレー背景 + 白文字
```

## 6. 抜け漏れているUX上の懸念点

### 6.1 情報量の圧倒感（Critical）

**問題**: 16ペルソナの分析結果を全て表示すると、ユーザーが情報に圧倒される。特に非技術者のターゲットユーザーにとって、技術系ペルソナの指摘は理解困難な可能性が高い。

**提案**:
- 総合サマリービューをデフォルト表示にし、「最も重要な改善提案TOP10」を優先度順で提示
- 各改善提案に「難易度タグ」（簡単/普通/専門家向け）を追加
- 「初心者向けビュー」と「詳細ビュー」の切り替え機能
- カテゴリごとのフィルタリング機能

### 6.2 分析開始前のペルソナ選択UX（High）

**問題**: 要件定義ではペルソナの有効/無効切り替えはペルソナ管理画面で行うが、分析開始時に実行するペルソナを選べるUIがない。16ペルソナ全実行は分析時間が長くなる。

**提案**:
- URL入力後、分析開始前にペルソナ選択のステップを追加（スキップ可能）
- 「クイック分析（主要5ペルソナ）」と「フル分析（全ペルソナ）」の2モード
- 最後に使った設定を記憶する機能

### 6.3 エラー状態のUXデザイン（High）

**問題**: 以下のエラーケースのUI設計が要件に含まれていない。
- 無効なURL
- スクレイピング失敗（robots.txt拒否、CAPTCHAページ等）
- AI API レート制限到達
- ネットワークエラー
- 特定ペルソナのみ分析失敗

**提案**:
- 各エラーに対して具体的なエラーメッセージとリカバリーアクションを設計
- 部分的失敗の場合は成功したペルソナの結果のみ表示し、失敗ペルソナにはリトライボタンを配置
- 「よくある問題と対処法」のヘルプセクション

### 6.4 初回訪問者の信頼性確保（Medium）

**問題**: 初回訪問者が「URLを入力して大丈夫か？」と不安に思う可能性。特に自社サイトのURLを入力する場合、データがどこに送信されるか懸念。

**提案**:
- 入力フォーム近くに「URLは分析のみに使用され、保存されません」の注記
- プライバシーポリシーへのリンク
- デモ分析をワンクリックで体験できる「デモを試す」ボタン
- 分析実績数や「OOサイトが分析されました」の社会的証明（将来的に）

### 6.5 分析結果の永続性と期待値管理（Medium）

**問題**: localStorageに保存される分析結果は、ブラウザのデータクリアで消失する。ユーザーがそれを認識していない可能性。

**提案**:
- 初回保存時に「結果はこのブラウザに保存されます。PDFエクスポートで永続化できます」の通知
- 重要な分析結果には「PDFでダウンロード」を目立つ位置に配置
- 共有URL生成を促すCTA

### 6.6 アクセシビリティの考慮不足（Medium）

**問題**: スコアのカラーコーディング、レーダーチャート等のビジュアル要素がスクリーンリーダーや色覚異常のユーザーに対応できるか。

**提案**:
- スコアのカラーに加えて、テキストラベル（優秀/良好/改善推奨/要改善）を必ず併記
- レーダーチャートにはテーブルビューの代替表示を用意
- すべてのインタラクティブ要素にキーボードナビゲーションを実装
- カラーコントラスト比はWCAG AAA基準（7:1以上）を目標

### 6.7 共有URLのプライバシーリスク（Medium）

**問題**: 分析結果のURLにはサイトURLが含まれる。社内サイトやステージング環境のURLが第三者に知られるリスク。

**提案**:
- 共有URL生成前に「共有される情報」の確認ダイアログを表示
- 共有URLに有効期限を設定するオプション（7日/30日/無期限）
- 「分析対象URLを隠す」オプション

### 6.8 分析結果の行動可能性（Low）

**問題**: 分析結果を見ても「具体的に何から手をつければよいか」が分かりにくい可能性。

**提案**:
- 「おすすめ改善ロードマップ」機能：改善提案を「今すぐ/今週中/今月中」に分類
- 各改善提案に「想定作業時間」の目安を追加
- チェックリスト形式で改善の進捗を管理できる機能（将来）

### 6.9 再分析のフロー（Low）

**問題**: 改善を実施した後の「再分析」フローが設計されていない。Before/After比較の具体的なUI仕様が不足。

**提案**:
- 分析結果ページに「再分析」ボタンを配置
- 同一URLの過去分析結果との自動比較ビュー
- スコアの推移グラフ（時系列表示）
- 改善された項目のハイライト表示
